# Roadmap

Capsule Wishes should grow in phases. The first priority is to make the emotional loop work before adding advanced intelligence or cloud features.

## Phase 0: Foundation

- Define product voice and UX principles.
- Replace default SwiftUI scaffold with first app shell.
- Add core domain models.
- Set up SwiftData persistence.
- Build a small design system for color, typography, spacing, and motion.

## Phase 1: MVP

- Narrative onboarding.
- Create wish capsule flow.
- Seal capsule animation.
- Capsule constellation / list screen.
- Capsule detail screen.
- Journal entry creation for:
  - strange thing;
  - small joy;
  - thought;
  - dream;
  - gratitude;
  - tiny step.
- Basic ritual selection.
- Local notifications for capsule opening.
- Capsule opening ceremony.
- Archive of opened capsules.

## Phase 2: Emotional Depth

- Mood before / after journal entries.
- Gentle reflection prompts.
- Wish theme extraction from repeated entries.
- Personal ritual resonance tracking.
- "What made today one percent lighter?" daily prompt.
- Better archive insights.

## Phase 3: Craft and Motion

- Rich capsule visuals.
- Subtle particle and glow system.
- Opening animation variations.
- Dynamic capsule states:
  - sleeping;
  - glowing;
  - pulsing;
  - ready to open;
  - opened.
- Accessibility and reduced motion support.

## Phase 4: Intelligence

- Optional AI-assisted wish clarification.
- Optional AI reflection summaries.
- Theme detection with user approval.
- Ritual recommendations based on previous resonance.
- Privacy-first settings for all AI features.

## Phase 5: Sync and Export

- iCloud sync.
- Local encrypted backup.
- Markdown / PDF export of opened capsules.
- Full data deletion and portability tools.

## Non-Goals for Early Versions

- Social feed.
- Public wishes.
- Competitive streaks.
- Medical or therapeutic claims.
- Heavy gamification.
- Guaranteed manifestation language.
